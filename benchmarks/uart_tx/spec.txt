Design: UART Transmitter (UART TX)
Protocol: 8N1 (8 data bits, No parity, 1 stop bit)

Inputs:
  clk          (1-bit) — System clock
  rst_n        (1-bit) — Active-low asynchronous reset
  tx_start     (1-bit) — Start transmission pulse (active high)
  data_in      (8-bit) — Parallel data to transmit
  baud_div     (16-bit) — Baud rate divisor (system_clk_freq / baud_rate)
                           Example: 50 MHz / 9600 = 5208 for 9600 baud

Outputs:
  tx_serial    (1-bit) — Serial data output
  tx_busy      (1-bit) — High during transmission, low when idle

Behavior:
  1. IDLE state: tx_serial = 1, tx_busy = 0
  2. When tx_start is asserted and tx_busy=0:
     - tx_busy goes high
     - Latch data_in into internal shift register
     - Start transmitting: start bit (0) + 8 data bits (LSB first) + stop bit (1)
  3. Each bit lasts (baud_div) clock cycles
  4. After stop bit completes, tx_busy goes low, return to IDLE
  5. If tx_start is asserted while tx_busy=1, ignore it (no buffering)

Reset behavior (rst_n=0):
  - tx_serial = 1
  - tx_busy = 0
  - Internal state = IDLE

Clock divider: use a counter that counts from 0 to baud_div-1
(toggle a baud_tick pulse at terminal count)

All flip-flops are positive edge-triggered (posedge clk).
